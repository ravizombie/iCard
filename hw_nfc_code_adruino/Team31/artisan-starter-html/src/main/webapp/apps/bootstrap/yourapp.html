This page is backed by the following artifacts in the following folders in your Artisan project:

/artisan-starter-html/src/main/webapp/apps/bootstrap/yourapp.html This HTML page. It imports and initializes the corresponding JavaScript.
/artisan-starter-html/src/main/webapp/apps/bootstrap/js/fetch.js This is the monolithic JS file which backs all of the tabs in this sample application.
/artisan-starter-html/src/main/webapp/apps/bootstrap/js/axeda.js This is the shared utility script which performs all actual communication with the Platform server.
/artisan-starter-scripts/src/main/groovy/tabs/YourApp.groovy The Groovy script which defines the Scripto Service that is invoked to read the Device status in JSON.


These files are all uploaded to the platform by using the Artisan uploader from the command line. This Uploader reads the "apc-metadata.xml" located in the root of your Artisan project. NOTE: You should only need to update this when you add new Custom Objects, as the HTML components will be automatically uploaded.

The flow of events within this page is as follows:

  1. This page "yourapp.html" is loaded by the Browser
  2. The script "/js/fetch.js" is included by the page
  3. The javascript backing this page is initialized by the "App.init();" call at the bottom of the page. After this, event listeners in the fetch.js script will trigger for this page!
  4. When this button (with an id of "getDeviceData") is clicked, a corresponding event handler in fetch.js calls the function Fetch.callScriptoFetchDeviceData()
  5. Fetch.callScriptoFetchDeviceData() in turn invokes the Script Service defined in /tabs/YourApp.groovy
  6. This service returns JSON representing the Device status, which is mapped to the HTML elements in this page by the Fetch.processDeviceJson() function
  7. This page then updates with the correct values.
